Course Management System

Hierarchical structure: Courses → Modules → Activities

Key Relationships

  • • Course → Modules (1:M) - Each course has multiple modules with weights
  • • Module → Activities (1:M) - Modules contain various activity types
  • • Activity → ActivityType (M:1) - Activities categorized by type
  • • Course ↔ GradeLevel (M:M) - Courses assigned to specific grades

ℹ️ Legend & Symbol Guide

Database Keys

PK
Primary Key
Unique identifier for each record
FK
Foreign Key
Reference to another table's primary key

Relationships

1 One - Single record
M Many - Multiple records
0..1 Optional - May or may not exist
━━━ Solid line - Required relationship
- - - Dashed line - Optional relationship

Entity Color Coding

Blue - Courses
Purple - Modules/Instructors
Pink - Activities
Amber - Types/Categories
Green - Students/Schedules
Cyan - Tracking/Pivot Tables
Red - Users/Security

Common Patterns

One-to-Many (1:M)
One parent has multiple children
Example: Course → Modules
Many-to-Many (M:M)
Multiple records linked via pivot table
Example: Course ↔ GradeLevel
Optional (0..1)
Relationship may or may not exist
Example: Schedule → Activity
